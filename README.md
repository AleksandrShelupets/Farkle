# FARKLE — консольна браузерна гра

Кубикова гра на ризик у темному термінальному стилі. Чистий HTML/CSS/JS,
**без збірки та залежностей** — працює офлайн.

## Запуск

Відкрийте `index.html` подвійним кліком (або перетягніть у браузер).
Жодного сервера чи `npm` не потрібно.

## Режими

- **Проти комп’ютера** — суперник з евристичною стратегією; 3 рівні складності.
- **Соло (рекорд)** — дійдіть до цілі за якомога **менше ходів**;
  найкращий результат зберігається в `localStorage`.

## Можливості

- **Збереження гри**: прогрес автоматично зберігається (`localStorage`) при
  перезавантаженні сторінки чи виході в меню — на головному екрані з’являється
  кнопка **«Продовжити гру»**, що відновлює партію з місця зупинки.
- Анімація кидка кубиків і звукові ефекти (через WebAudio; вимикаються в налаштуваннях).
- Авто-вибір очкових кубиків (клавіша `A`) і підсвічування тих, що дають очки.
- Індикатор ризику «прогоріти» для наступного кидка.
- **Налаштування**: ціль очок (5000 / 10000 / власна), правило «зайти з 500»,
  складність AI, звук, тема (зелена / янтарна / синя).
- **Статистика**: ігри, перемоги проти AI, к-сть Farkle, очки втрачені через Farkle, найкращий хід, рекорд ходів.
- **Ім’я гравця** (в налаштуваннях) + **спільний рейтинг** усіх гравців (3 окремі
  таблиці: перемоги проти AI / найкращий хід / соло-ходи). Без БД — серверний
  JSON-файл через PHP; ім’я перевіряється на унікальність. Клавіша `R`.
- **Дві мови** інтерфейсу: українська / English (перемикач у налаштуваннях,
  застосовується одразу; зберігається в `localStorage`).
- Екран перемоги; доступність (ARIA, фокус, `prefers-reduced-motion`); адаптив; PWA.
- Версія гри показується у футері меню (константа `APP_VERSION` у `js/main.js`).

## Правила (стисло)

Кидаєте 6 кубиків, після кожного кидка відкладаєте ≥1 очковий кубик, потім або
кидаєте далі (ризик), або **банкуєте** очки ходу. Кидок без очок — **FARKLE**,
очки ходу згоряють. Усі 6 відкладені — **гарячі кубики**, берете 6 нових.
Перемога — перший до цілі (суперник отримує останній хід).

| Комбінація | Очки |
|---|---|
| Одна 1 / Одна 5 | 100 / 50 |
| Три 1 | 1000 |
| Три 2/3/4/5/6 | 200/300/400/500/600 |
| Чотири / П’ять / Шість однакових | 1000 / 2000 / 3000 |
| Стріт 1-2-3-4-5-6 | 1500 |
| Три пари | 1500 |
| Дві трійки | 2500 |

## Керування

- Миша: клік по кубику — вибрати/зняти; кнопки внизу.
- Клавіатура у грі: **1–6** кубик · **A** вибрати всі очкові · **Enter** кинути /
  кинути ще · **B** банк · **Esc** меню.
- Клавіатура в меню: **1** / **2** режими · **S** налаштування · **T** статистика ·
  **R** рейтинг · **H** правила.

## Структура

```
index.html              розмітка + порядок скриптів
css/styles.css          темна тема + теми (green/amber/blue), адаптив, a11y
js/scoring.js           підрахунок очок + ймовірність Farkle (чисті функції)
js/sound.js             звукові ефекти (WebAudio, без файлів)
js/store.js             налаштування + статистика + ім'я/clientId + мова + збережена гра (localStorage)
js/i18n.js              словник uk/en + t(key, params)
js/net.js               AJAX до рейтингу (api/leaderboard.php)
js/ai.js                стратегія комп’ютера + рівні складності
js/game.js              стан і машина ходу (+ правило «зайти з 500»)
js/ui.js                рендер DOM, лог, ввід, модалки, теми (уся текстівка)
js/main.js              контролер (зв’язує все, веде хід AI, статистика, рейтинг)
js/tests.js             самотест підрахунку
api/leaderboard.php     серверний рейтинг (list/register/submit) — JSON-файл, без БД
data/leaderboard.json   сховище рейтингу (має бути доступним для запису PHP)
fonts/farkle-logo.woff2  сабсет шрифта для лого (псевдографіка)
manifest.webmanifest    PWA-маніфест
sw.js                   service worker (офлайн-кеш; /api/ завжди з мережі)
favicon.svg             іконка
```

## Рейтинг (спільний, без БД)

Імена та показники зберігаються у `data/leaderboard.json` через `api/leaderboard.php`
(жодної БД). Вимоги до хостингу:

- **PHP** (7.x/8.x), бажано з `mbstring`.
- Тека `data/` має бути **доступна для запису** вебсервером (напр. `chmod 775 data`).
- `data/.htaccess` закриває прямий веб-доступ до файлу (Apache/LiteSpeed). На nginx
  закрийте `/data/` у конфігу окремо.

Унікальність імені без авторизації: кожен браузер має випадковий `clientId`, що стає
«власником» імені. Те саме ім’я з тим самим браузером — це ви; інший браузер отримає
«ім’я зайняте». Офлайн / `file://` рейтинг недоступний — гра працює, ім’я зберігається
лише локально.

## Самотест

Відкрийте `index.html?test=1` — у журналі з’явиться результат перевірки
функцій підрахунку (та деталі у консолі браузера).

## PWA / встановлення

Гра й так працює офлайн із `file://`. Щоб встановити як застосунок (service
worker), подайте теку статичним сервером, напр. `npx serve` чи
`python -m http.server`, і відкрийте через `http://localhost:…`.
